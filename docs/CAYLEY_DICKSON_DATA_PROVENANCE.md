## Key Findings

### Confirmed by cross-validation

1. **Strut table (C-454):** De Marrais's published strut table for 7 sedenion
   box-kites matches our Rust computation exactly (all 42 assessors, all 21
   strut pairs).

2. **Inner XOR property:** low ^ high = 8 + strut_constant for all assessors.

3. **NATO trips:** Zigzag L-indices always form an octonion associative triple.

4. **Lattice mapping:** All dims 256-2048 embed into 8D lattice (NOT
   log2(dim)-dimensional as initially hypothesized). This suggests the
   octonion sub-algebra (8D) provides the fundamental lattice structure.

5. **Nested-tuple parser:** Correctly handles Cayley-Dickson doubling tree
   representation for dims 4 through 2048.

### Refuted by cross-validation

1. **256D associativity (C-456):** CSV claims all 125 tested triples are
   associative. Our Rust finds 4/50 non-associative triples involving
   high-index basis elements (e_128, e_64, e_32, e_192). The CSV is wrong.

### Novel negative results

1. **E8 lattice-ZD differences (C-455):** Lattice differences between
   ZD-adjacent pairs have norm-squared values of 4, 6, 8, 10, 12, 14, 18
   but NEVER 2. E8 roots would appear at norm-squared = 2. The ZD adjacency
   structure does not map to E8 root vectors under this lattice embedding.
   Minimum separation is |d|^2 = 4 (D4/D8 lattice vectors).

## Data Quality Assessment

- **PDFs:** Authentic de Marrais manuscripts. Strut table data verified.
- **Lattice CSVs:** Correct format, parse cleanly, consistent across dims.
  Lattice coordinates include {-1, 0, 1} (not just {-1, +1} as hypothesized).
- **Adjacency CSVs:** Multiple incompatible representations of "adjacency"
  exist (sparse vs dense, different node labelings). Treat with caution.
- **Associativity CSV:** Contains errors. Do not treat as ground truth.
- **Comparison/qualitative CSVs:** AI-generated speculation. No numerical
  content to verify.
